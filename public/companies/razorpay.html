How Razorpay Interviews for AI/ML Roles

Payments and fintech infrastructure company; ML/DS roles focus on fraud/risk detection, payment success optimization, real-time ranking, and forecasting. The loop weights production-grade coding (Python/SQL/PySpark) alongside ML depth and system design.

The Interview Loop

  1. Recruiter screen — ~30-minute call on background, experience, motivation, and role/culture alignment.
  2. Technical assessment — Coding via platform or video: algorithms, Python, and ML concepts, with a production-implementation lens.
  3. Coding (Python/SQL/PySpark) — Progressively harder problems across Python, SQL, and PySpark.
  4. System design + ML depth — ML knowledge plus designing systems such as a real-time ranking engine for payment gateways; metrics like AUC/ROC, precision, recall.
  5. Hiring manager — Cultural fit, levelling, mentoring, communication, and long-term vision.

What They Weight

  • Production-quality code and data engineering (PySpark) fluency
  • Real-time, low-latency ML system design for payments
  • Solid grasp of classification metrics for fraud/risk
  • Ownership and ability to ship into production

Focus Your Prep On

  • Python, SQL, and PySpark coding
  • Algorithms and data structures
  • Classification models and metrics (AUC, ROC, precision, recall)
  • Real-time ranking / scoring system design
  • Fraud and risk modeling context

Prep angle: Sharpen Python/SQL/PySpark and be able to design a real-time scoring or ranking system for payments; revise classification metrics and prepare fraud/risk framing for your project stories.
